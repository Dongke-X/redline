— 三个核心场景

三种 HTML，一种 review 方式。

幻灯片、文档、网页——Redline 三种都能注入。随改随预览、改完导 PDF。同一套工作流、同一份导出 schema。

01 · 幻灯片

HTML 幻灯片

Reveal.js、deck-stage、纯 HTML slides——Redline 自动识别每页，按页编辑、按元素改写。导出时每个改动都标注归属第几页。

"section": "slide-3", "op": "text", "before": "Q3 数据指标", "after": "Q3 业务结果"
02 · 文档

HTML 报告 / 文档

agent 写的长文档——RFC、白皮书、设计 brief。按章节标注、按段落批注、标出前后矛盾。agent 端拿到的是按章节归类的反馈。

"section": "3.2", "perSection": [ { note: "补引用来源" }, { note: "术语太多，说人话" } ]
03 · 网页

HTML 网页 / Landing / UI

最丰富的场景：agent 写的 landing、dashboard、原型。改文案、拖元素、画区域、贴参考截图——所有动作打包成一个 ZIP 给 agent。

"edits": [ … 3 ], "annotations": [ … 2 ], "attachments": [ … 1 ]
— 不只是标注

随改随预览。一键导出 PDF。

Redline 改的是真实 DOM，每改一下立刻看到。

所见即所得

不用等"重新生成"。打字 → 立刻看到。拖元素 → 实时跟手。Save → 跟你看到的一模一样。同一份 DOM，同一份渲染。

Space + P

矢量 PDF

文字可选中复制，任意缩放都清晰，链接保留可点。给客户、打印、邮件附件都能用——不是一张死截图。

Shift + P

长截图 PDF

整个页面装一张超长图里。不分页、不在尴尬的地方断开。最适合展示完整 landing 或者一个 deck 从头到尾的全貌。

— 问题

Agent 写完 HTML，
你怎么告诉它哪里不对？

跟 Claude / Codex / Gemini 协作生成 HTML 幻灯片、文档、网页都很爽——直到你想反馈"第 4 页那句要改、第 3 张卡片颜色不对、那段 padding 多了 4px"。

发截图

Agent 看的是像素，不是 DOM

你贴张 PNG 说"这个标题改一下"，agent 盯着图看不出 selector，只能 grep 猜在哪个文件、哪段 HTML。猜对就猜对，猜错就重来。

复制 outerHTML

你成了人肉 querySelector

打开 DevTools 找元素、拷 outerHTML、粘到对话窗、再描述要改成什么。一处这样、七处也这样。一页有 12 个改动？打字打到天亮。

打字描述

"第 3 页 bullet 2、第 5 页 hero 文案……"

写 50 行 prompt 把所有改动罗列一遍。agent 漏一个、改错一个，下一轮你又写 50 行清单去修。每一轮都是新负担。

— 工作原理

三步走，不用注册。

点工具栏图标，标注 / 改写，下载 ZIP。Agent 读 JSON，把改动应用到源码里。

第 01 步

让编辑器进到页面

两条路径都可以：对 Claude 说 "prep this html" 让 skill 自动注入；或者装了 Chrome 扩展的话，直接点工具栏图标。

第 02 步

标注一遍

改文字、画框、贴截图——三种用法下一区分别详解。

第 03 步

下载 ZIP

一个文件搞定：session.json + Markdown 摘要 + 所有截图。丢给 Claude / Codex 让它改。

— 功能

为 AI 协作时代而设计。

每个改动都带稳定的 CSS selector。Agent 不用瞎猜改动该落在哪。

原地改写文字

点任何标题、按钮、段落，直接打字。改前 / 改后 都打包给 agent。

区域标注

拖个框圈住任何东西，加备注。自动编号、按顺序、可导出。

截图捕获

直接粘贴截图到反馈面板。导出 ZIP 时变成单独的图片文件。

CSS Selector 映射

每个改动都带稳定 selector。Agent 直接改源码，不用当侦探。

源码感知导出

JSON + Markdown + 图片文件。schema 文档化、稳定。Claude Code Skill 已对接。

本地优先 · 隐私

没有服务器、没有埋点、没有第三方 SDK。所有数据都留在你浏览器里。

— 配套 skill

装 skill 就够。
不用 Chrome 扩展也能在浏览器里编辑。

v0.1.0 自带 prepare.mjs：跟 Claude 说"prep 这个 HTML"，skill 自动把编辑器注入到你的文件里。浏览器打开 → review → save → skill 把改动 patch 回源 HTML。闭环，不依赖扩展（扩展仍可用于 review 远程 URL）。

01 · 注入

prepare.mjs — 给 HTML 预埋编辑器

在你的 HTML 里加 <script src="redline.js">，浏览器打开自动加载。--inline 把 bundle 嵌进去，--remove 一键还原。idempotent + 自动备份。

02 · 应用

apply.mjs — 把反馈 patch 回源

读 ZIP，按多策略（id / cssPath / contentHash）解析 selector，批量执行 text / move / scale / hide / delete。区域标注 + 叙事性反馈则跟你逐条对齐。

安装 · 一行命令
# 在 redline 仓库目录下
npm run install:skill

# 或者手动拷贝
cp -r skill/ ~/.claude/skills/redline/
使用 · 在 Claude Code 里
你: "给 report.html 加 redline"
claude: 跑 prepare.mjs report.html
claude: "浏览器打开它，按 F 反馈"

# 浏览器里 Save 之后：
你: "应用 redline 的反馈"
claude: 读 ~/.redline/sessions/…
claude: 7 处改动写回 report.html
claude: "还有 2 条标注需要你判断 ——"

— 输出格式

一套 schema，一个文件，长期稳定。

session.json 大概长这样。版本化、有文档、对 agent 友好。

session.json
// 已精简便于阅读 — 完整 schema 见仓库 schema/ 目录
{
  "schemaVersion": "1.0",
  "appMode": "deck",
  "capturedAt": "2026-05-10T14:23:11Z",
  "page": { "url": "file:///~/agent-out/q3-deck.html", "title": "Q3 业务回顾" },

  "edits": [
    {
      "op": "text",
      "section": "slide-3",
      "selector": {
        "cssPath": "section.slide:nth-of-type(3) > h1",
        "contentHash": { "sample": "Q3 数据指标", "tag": "h1" }
        // + id / fbId / tag — cssPath 变了还能靠 contentHash 兜底
      },
      "before": "Q3 数据指标",
      "after":  "Q3 业务结果 — 哪些做对了"
    }
  ],

  "annotations": [
    {
      "id": "anno-7f3a",
      "type": "region",
      "secId": "slide-5",
      "rectPct": { "x": 12, "y": 48, "w": 35, "h": 22 },
      "note": "这页图表太密 — 换成柱状图",
      "image": { "filename": "session-anno-7f3a.png" }
    }
  ],

  "feedback": { "global": "整体节奏：开场收紧，第 5 页换图" }
}
"Agent 不用猜你想表达什么。它读 selector，找到元素，把改动写回去。" — 为什么"源码感知"比"像素级截图"更重要。
