— Three core scenarios

Three HTML formats. One way to review.

Slides, docs, web pages — Redline injects into all three. Edit live, see changes instantly, export PDF when done. Same workflow, same export schema.

01 · DECK

HTML slide decks

Reveal.js, deck-stage, plain HTML slides — Redline detects each slide, lets you edit per-slide and per-element. The export tags every change with its slide number.

"section": "slide-3", "op": "text", "before": "Q3 metrics", "after": "Q3 results"
02 · DOC

HTML reports & docs

Long-form HTML output from the agent — RFCs, white papers, design briefs. Annotate sections, mark paragraphs, flag inconsistencies. The agent sees structured per-section feedback.

"section": "3.2", "perSection": [ { note: "cite source" }, { note: "simplify jargon" } ]
03 · WEB

HTML landing pages & UIs

The full mix: agent-generated landings, dashboards, prototypes. Edit copy, drag elements, annotate regions, paste reference screenshots. Everything ships in one bundle.

"edits": [ … 3 ], "annotations": [ … 2 ], "attachments": [ … 1 ]

— The problem

The agent shipped HTML.
Now you have to tell it what's wrong.

Vibe-coding with Claude / Codex / Gemini is great until you need to give feedback. Slides, docs, landing pages — all generated, all almost-right, all missing that sentence on page 4 and that color on the third card.

Send a screenshot

The agent sees pixels, not the DOM.

You paste a PNG saying "change this headline." The agent stares at the image, can't read your selector, and writes a regex grep to guess which file it's in.

Copy outerHTML

You become a human querySelector.

Open DevTools, find the element, copy outerHTML, paste into chat, explain what to change. Now do that 7 more times for the rest of the page.

Type out a list

"Slide 3 bullet 2, slide 5 hero, …"

50-line prompt describing every change. Agent misses one, edits another wrong. Next round you write another 50-line prompt to clean up.

— How it works

Three steps. No accounts.

Click the toolbar icon, mark it up, download a ZIP. The agent reads the JSON and applies your edits to source.

Step 01

Get the editor in

Two paths, same result: tell Claude "prep this html" to inject via skill, or click the toolbar icon if you've installed the extension.

Step 02

Mark it up

Edit text, draw regions, paste screenshots — three flavors detailed in the next section.

Step 03

Download ZIP

One file: session.json, a Markdown summary, and all screenshots. Hand it to Claude / Codex.

— Features

Built for the agent era.

Every capture ships with a stable CSS selector. Your agent doesn't have to guess where the change goes.

Inline text editing

Click any heading, button, paragraph — type the new copy. Before/after both ship to the agent.

Region annotations

Drag a box around anything. Add a note. Numbered, ordered, exportable.

Screenshot capture

Paste screenshots straight into the panel. They become real image files inside the export ZIP.

CSS-selector mapping

Every edit ships with a stable selector. Agents patch source HTML without playing detective.

Source-aware export

JSON + Markdown + image files. Schema is documented and stable. Agents have a Skill for it.

Local-first privacy

No servers, no analytics, no third-party SDKs. Everything stays in your browser.

— Companion skill

The skill alone is enough.
Inject the editor without a Chrome extension.

v0.1.0 ships prepare.mjs: tell Claude "prep this HTML," skill injects the editor into your file. Open in any browser → review → save → skill applies feedback to source. Closed loop, no extension required (but the extension is still there if you want to review live URLs).

01 · PREPARE

prepare.mjs — inject the editor

Adds <script src="redline.js"> to your HTML so the editor auto-loads in any browser. --inline to embed the bundle directly. --remove to clean. Idempotent + auto-backup.

02 · APPLY

apply.mjs — patch back to source

Reads the saved ZIP, resolves selectors with multi-strategy fallback (id / cssPath / contentHash), applies text / move / scale / hide / delete in batch. Region annotations + narrative feedback get walked through with you.

INSTALL · ONE-LINER
# in the redline repo
npm run install:skill

# or copy manually
cp -r skill/ ~/.claude/skills/redline/
USE · IN CLAUDE CODE
you: "prep report.html for review"
claude: runs prepare.mjs report.html
claude: "open it in browser, F to feedback"

# after you Save in browser:
you: "apply the redline feedback"
claude: reads ~/.redline/sessions/…
claude: applies 7 edits to report.html
claude: "2 annotations need your call —"
Native to Claude Code. Bundle is portable to any agent.
NATIVE SKILL
Claude Code

PORTABLE
Codex · OpenClaw · Hermes · any file-reading agent
Drop the unzipped session into your project. Then send the agent this one-liner — schema is self-describing, no plugin needed:
read session.json. for each edit, find selector.cssPath (fallback contentHash) and replace before with after. walk me through annotations[].
Browser collects → ZIP exports → any agent applies. Closed loop.
— Output

One schema. One file. Stable.

Here's the shape of session.json. Versioned, documented, agent-friendly.

session.json
// trimmed for readability — see schema/ in repo for full spec
{
  "schemaVersion": "1.0",
  "appMode": "deck",
  "capturedAt": "2026-05-10T14:23:11Z",
  "page": { "url": "file:///~/agent-out/q3-deck.html", "title": "Q3 review" },

  "edits": [
    {
      "op": "text",
      "section": "slide-3",
      "selector": {
        "cssPath": "section.slide:nth-of-type(3) > h1",
        "contentHash": { "sample": "Q3 metrics", "tag": "h1" }
        // + id / fbId / tag — multi-strategy fallback if cssPath shifts
      },
      "before": "Q3 metrics",
      "after":  "Q3 results — what worked"
    }
  ],

  "annotations": [
    {
      "id": "anno-7f3a",
      "type": "region",
      "secId": "slide-5",
      "rectPct": { "x": 12, "y": 48, "w": 35, "h": 22 },
      "note": "chart too dense — switch to bar",
      "image": { "filename": "session-anno-7f3a.png" }
    }
  ],

  "feedback": { "global": "deck overall: tighter intro, swap chart on slide 5" }
}
